## Project preferences

- Be **very concise** in explanations. Prefer short answers and avoid repeating context.
- **Avoid unnecessary tool calls** (file reads, searches, lints, shells) when the answer is clear from the prompt.
- When editing, **only touch the files and lines explicitly requested** by the user.
- Prefer **minimal diffs** and do not refactor unrelated code unless the user asks.
- Do **not restate unchanged code**; only show or describe what actually changed.
- Default to **no additional commentary** after successful edits unless the user requested detailed explanation.
